# Production Deployment Guide - ESportNews Go Backend

## Overview

This guide covers deploying the ESportNews Go backend to a production VPS using Docker Compose.

**Expected Duration:** 30-45 minutes
**Required VPS Specs:**
- OS: Ubuntu 22.04 LTS
- CPU: 2+ cores
- RAM: 4GB minimum
- Storage: 50GB SSD
- Network: 100+ Mbps

---

## Phase 1: VPS Preparation (10 minutes)

### 1.1 Initial SSH Connection

```bash
# Connect to your VPS
ssh root@your-vps-ip

# Update system packages
apt update && apt upgrade -y

# Install essential tools
apt install -y curl wget git htop nano jq
```

### 1.2 Install Docker

```bash
# Download and install Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Verify installation
docker --version
docker run hello-world

# Add current user to docker group (optional)
sudo usermod -aG docker ${USER}
```

### 1.3 Install Docker Compose

```bash
# Download Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# Make executable
sudo chmod +x /usr/local/bin/docker-compose

# Verify installation
docker-compose --version
```

### 1.4 Install Dependencies

```bash
# Install Node.js (for npm)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
apt install -y nodejs

# Install certbot for SSL
apt install -y certbot python3-certbot-nginx

# Verify
node --version && npm --version
```

## Phase 2: Application Setup (10 minutes)

### 2.1 Clone Repository

```bash
# Navigate to appropriate directory
cd /opt

# Clone the repository
git clone https://github.com/esportnews/esportnews.git
cd esportnews

# Set permissions
sudo chown -R $USER:$USER .
```

### 2.2 Create Environment Configuration

```bash
# Create .env file for production
cat > .env.production << 'EOF'
# Environment
ENVIRONMENT=production

# Database
DB_USER=postgres
DB_PASSWORD=your_secure_password_here
DB_NAME=esportnews

# Security (generate with: openssl rand -base64 32)
JWT_SECRET=your_jwt_secret_here

# External APIs
PANDASCORE_API_KEY=your_pandascore_api_key
SPORTDEVS_API_KEY=your_sportdevs_api_key

# Server Configuration
SERVER_PORT=4000
LOG_LEVEL=info

# Performance Tuning
CACHE_TTL=3600
DB_MAX_CONNECTIONS=50

# CORS Configuration
CORS_ORIGINS=https://yourdomain.com,https://www.yourdomain.com

# Email (optional)
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=your_email@example.com
SMTP_PASSWORD=your_email_password
EOF

# Set secure permissions
chmod 600 .env.production
```

### 2.3 Generate Secure Secrets

```bash
# Generate JWT Secret
JWT_SECRET=$(openssl rand -base64 32)
echo "JWT_SECRET=$JWT_SECRET"

# Generate strong database password
DB_PASSWORD=$(openssl rand -base64 24)
echo "DB_PASSWORD=$DB_PASSWORD"

# Update .env.production with these values
```

### 2.4 Prepare Nginx Configuration

```bash
# Create nginx configuration directory
mkdir -p nginx/conf.d

# Create main nginx config
cat > nginx.conf << 'EOF'
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    client_max_body_size 20M;

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript
               application/x-javascript application/xml+rss
               application/json application/javascript;

    # Rate limiting
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=100r/m;
    limit_req_zone $binary_remote_addr zone=auth_limit:10m rate=10r/m;

    # Upstream backend
    upstream esportnews_backend {
        server backend:4000 max_fails=3 fail_timeout=30s;
    }

    server {
        listen 80;
        server_name _;

        location /.well-known/acme-challenge/ {
            root /var/www/certbot;
        }

        location / {
            return 301 https://$host$request_uri;
        }
    }

    # Include SSL configuration (generated by certbot)
    include /etc/nginx/conf.d/*.conf;
}
EOF
```

## Phase 3: SSL/TLS Setup (10 minutes)

### 3.1 Configure SSL with Let's Encrypt

```bash
# Create certbot directories
mkdir -p certs

# Stop services temporarily (if running)
docker-compose down 2>/dev/null || true

# Get certificate (replace with your domain)
certbot certonly --standalone -d yourdomain.com -d www.yourdomain.com

# Copy certificates to project directory
sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem certs/
sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem certs/
sudo chown $USER:$USER certs/*
```

### 3.2 Configure Nginx SSL

```bash
# Create HTTPS nginx configuration
cat >> nginx.conf << 'EOF'
    server {
        listen 443 ssl http2;
        server_name yourdomain.com www.yourdomain.com;

        ssl_certificate /etc/nginx/certs/fullchain.pem;
        ssl_certificate_key /etc/nginx/certs/privkey.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        ssl_prefer_server_ciphers on;

        # Security headers
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Referrer-Policy "no-referrer-when-downgrade" always;

        location /health {
            proxy_pass http://esportnews_backend;
            access_log off;
        }

        location /api/auth {
            limit_req zone=auth_limit burst=5 nodelay;
            proxy_pass http://esportnews_backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_read_timeout 60s;
        }

        location / {
            limit_req zone=api_limit burst=20 nodelay;
            proxy_pass http://esportnews_backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_cache_bypass $http_upgrade;
            proxy_read_timeout 60s;
        }
    }
EOF
```

### 3.3 Auto-Renewal Setup

```bash
# Create renewal script
cat > /usr/local/bin/renew-certs.sh << 'EOF'
#!/bin/bash
cd /opt/esportnews
certbot renew --quiet
cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem certs/
cp /etc/letsencrypt/live/yourdomain.com/privkey.pem certs/
docker-compose -f docker-compose.prod.yml restart nginx
EOF

chmod +x /usr/local/bin/renew-certs.sh

# Add to crontab for automatic renewal
(crontab -l 2>/dev/null; echo "0 0 * * * /usr/local/bin/renew-certs.sh") | crontab -
```

## Phase 4: Database Migration (5 minutes)

### 4.1 Create Database Backup

```bash
# Create backup directory
mkdir -p /backups

# If migrating from Supabase, create dump
pg_dump -h db.olybccviffjiqjmnsysn.supabase.co \
        -U postgres \
        -d postgres \
        -f /backups/supabase_backup_$(date +%Y%m%d_%H%M%S).sql

# Compress backup
gzip /backups/supabase_backup_*.sql
```

### 4.2 Start Services

```bash
# Start with production config
export $(cat .env.production | xargs)
docker-compose -f docker-compose.prod.yml up -d

# Wait for services to be healthy
sleep 30

# Check status
docker-compose -f docker-compose.prod.yml ps
```

### 4.3 Initialize Database

```bash
# Create database and run migrations
docker-compose -f docker-compose.prod.yml exec backend \
    psql -h postgres -U postgres -d esportnews \
    -f /docker-entrypoint-initdb.d/schema.sql

# Restore data if migrating from Supabase
docker-compose -f docker-compose.prod.yml exec -T postgres \
    psql -U postgres -d esportnews < /backups/supabase_backup.sql.gz
```

## Phase 7: Maintenance & Operations

### 7.1 Regular Backups

```bash
# Create backup script
cat > /usr/local/bin/backup-db.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/backups"
DATE=$(date +%Y%m%d_%H%M%S)

docker-compose -f /opt/esportnews/docker-compose.prod.yml exec -T postgres \
    pg_dump -U postgres -d esportnews | gzip > $BACKUP_DIR/esportnews_$DATE.sql.gz

# Keep only last 7 days of backups
find $BACKUP_DIR -name "esportnews_*.sql.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR/esportnews_$DATE.sql.gz"
EOF

chmod +x /usr/local/bin/backup-db.sh

# Schedule daily backups at 2 AM
(crontab -l 2>/dev/null; echo "0 2 * * * /usr/local/bin/backup-db.sh") | crontab -
```

### 7.2 Log Management

## Troubleshooting

### Issue: "Connection refused" on port 80/443

```bash
# Check port availability
sudo lsof -i :80
sudo lsof -i :443

# Kill existing process
sudo kill -9 <PID>

# Verify firewall
sudo ufw status
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

### Issue: Database won't start

```bash
# Check PostgreSQL logs
docker-compose logs postgres

# Verify data directory permissions
sudo chown 999:999 /var/lib/docker/volumes/postgres_data_prod/_data

# Restart PostgreSQL
docker-compose restart postgres
```

### Issue: High CPU/Memory usage

```bash
# Check container stats
docker stats

# Reduce worker processes in nginx
worker_processes 2;

# Increase cache cleanup in Redis
# Adjust maxmemory-policy in redis command

# Restart services
docker-compose restart
```

### Issue: SSL certificate not working

```bash
# Verify certificate
openssl x509 -in certs/fullchain.pem -text -noout

# Check certificate validity
certbot certificates

# Renew manually
certbot renew --force-renewal

# Restart nginx
docker-compose restart nginx
```
